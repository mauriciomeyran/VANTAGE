## SECCIÓN 5 — FASES Y LÓGICA DEL SISTEMA

### F0 — CAPTURA DE VACANTE
**Trigger:** Vacante nueva detectada (cualquier fuente)
**Acciones:**
1. Verificar URL activa (fetch manual o herramienta). Si 404 → marcar `Descartada` + nota "URL inactiva".
2. Extraer: Empresa, Rol, URL, Fuente, Fecha de entrada, Job ID.
3. Generar Job ID: `[EMPRESA]-[YYYYMM]-[INICIALES_ROL]` (ej: `NIKE-202503-MVM`)
4. Verificar duplicado: si Job ID ya existe en Tracker → STOP, no agregar.
5. Crear entrada en Pipeline con status `🆕 Nueva`.

**Gate F0 → F1:** Vacante activa + no duplicada.

---

### F0.5 — ENRIQUECIMIENTO EXTERNO (CONDICIONAL)
**Trigger:** JD tiene < 300 palabras O contiene términos no inferibles del sector.
**Herramientas:**
- **Perplexity:** Contexto de empresa (cultura, recent news, headcount, posición de mercado)
- **Gemini:** Keywords sectoriales complementarias, sinónimos ATS del sector

**Reglas:**
- NO reemplazar JD original — enriquecimiento es addendum, no sustitución.
- NO inventar datos. Si Perplexity no retorna nada confiable → marcar como "Sin datos externos".
- Etiquetar cada dato externo con `[FUENTE: Perplexity]` o `[FUENTE: Gemini]`.
- Mostrar diferencias entre JD original y datos externos antes de proceder.

**Output:** Bloque de contexto adjunto al JD original en Notion, marcado como F0.5.

**Gate F0.5 → F1:** Datos externos integrados y marcados. Aprobación manual antes de continuar.

### F2 — FIT SCORE (ATS)
**Pesos:**

| Criterio | Peso |
|---------|------|
| Keywords obligatorias presentes | 40% |
| Keywords deseables presentes | 15% |
| Años de experiencia requeridos vs reales | 15% |
| Industria/sector match | 15% |
| Nivel del rol (Sr/Coordinator/Manager/Director) vs historial | 10% |
| Idioma del CV vs JD | 5% |

**Cálculo:**
- Cada keyword obligatoria: 40/N puntos (N = total obligatorias)
- Cada keyword deseable: 15/M puntos (M = total deseables)
- Años: match exacto = 15; ±2 años = 10; >2 años gap = 5; más del requerido = 15
- Industria: match directo = 15; transferible = 8; diferente = 0
- Nivel: match = 10; aspiracional (1 nivel arriba) = 7; sobre-calificado = 5; 2+ niveles = 0
- Idioma: match = 5; diferente = 0

**Gates:**
- Score < 40 → `Descartada` automáticamente (no es viable sin mentir)
- Score 40-59 → `En revisión` — requiere decisión manual + nota de riesgo
- Score 60-79 → `Apta para optimizar` — proceder con advertencias de gap
- Score ≥ 80 → `Alta compatibilidad` — proceder directo

### F3 — GAP ANALYSIS
**Para cada gap identificado:**
```
GAP: [keyword/skill faltante]
SEVERIDAD: CRÍTICO / MODERADO / MENOR
WORKAROUND: [experiencia del CV que cubre parcialmente]
RIESGO RESIDUAL: [lo que no queda cubierto]
ACCIÓN: CUBRIR_CON_EXPERIENCIA / MENCIONAR_EN_COVER / ACEPTAR_GAP / DESCARTAR
```

**Gaps que nunca se pueden fabricar:**
- Headcount management (si no existe en ningún rol → no agregar)
- Idiomas no documentados
- Software no usado
- Industrias sin ningún punto de contacto

**Gaps que SÍ se pueden resolver:**
- Terminología diferente para misma habilidad → reframear
- Experiencia en industria adyacente → analogía explícita
- Escala diferente → contextualizar con métricas propias
