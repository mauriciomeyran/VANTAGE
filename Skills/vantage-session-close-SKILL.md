1. INVENTORY: El operador declara directamente si hubo cambios en la sesión[span_16](start_span)[span_16](end_span). 2. CENSUS: Si el operador declaró cambios de ID, se requiere el output de `generate_census.py` ejecutado localmente[span_19](start_span)[span_19](end_span). Sin este output, marcar ticket(s) como `Blocked-Census`[span_20](start_span)[span_20](end_span).

4. VERIFY: El operador ejecutará `python3 scripts/verify_versions.py --check` localmente y pegará el output[span_23](start_span)[span_23](end_span). Claude valida el string `PASS`[span_24](start_span)[span_24](end_span). Prohibido hacer llamadas MCP a los 7 documentos fundacionales (Kernel, Manual, Canon, SP, Aliases, Changelog, Census) para verificar[span_25](start_span)[span_25](end_span).

Este comando ya escribió la versión maestra del Changelog en los 6 documentos restantes directamente en Notion — housekeeping mecánico, exento de `APROBAR_WRITE` (que queda reservado para modificaciones de contenido, código o IDs). Claude no avanza al paso 5 sin recibir este output de texto en la sesión actual[span_27](start_span)[span_27](end_span).
5. SUMMARY: Sintetizar en máximo 2 líneas el trabajo crítico completado y los pendientes urgentes[span_28](start_span)[span_28](end_span).
